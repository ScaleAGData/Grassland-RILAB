# FCOVERModel: Biophysical Parameter Estimation

## 1. Overview
The `FCOVERModel` repository is a specialized component of the **ScaleAgData Grassland RILAB** pipeline. Its primary purpose is to train and validate machine learning models that estimate the **Fraction of Vegetation Cover (FCOVER)** using gap-filled **CropSAR2D** data.

FCOVER is a critical biophysical variable that quantifies the spatial extent of green vegetation. In this project, it serves as a key input for estimating Gross Primary Production (GPP) and monitoring grassland health.

## 2. Methodology
This repository implements a robust training workflow designed for high-resolution agricultural monitoring:

*   **Multi-Algorithm Training:** Simultaneously trains and compares **Random Forest (RF)** and **XGBoost (XGB)** regressors.
*   **Gap-Filled Data Integration:** Leverages **CropSAR2D** datasets which fuse Sentinel-1 (Radar) and Sentinel-2 (Optical) data to overcome cloud-cover limitations.
*   **Iterative Optimization:** The script performs up to **1,000 iterations**, tracking validation metrics to ensure the final model is the most stable and accurate version possible.
*   **Feature Engineering:**
    *   Integration of vegetation indices (FCOVER).
    *   Cyclical temporal encoding ($DOY_{sin}$, $DOY_{cos}$) to capture phenological seasonality.

## 3. Repository Structure
The following files are essential to the FCOVER modeling process:

*   `FCOVER4GPP_RF_XGB_1.py`: The main execution script for model training and optimization.
*   `Input_IFAPA_GPP_cropsar2D_3vars.csv`: (Directory) Should contain `Input_IFAPA_GPP_cropsar2D_3vars.csv` or similar synchronized datasets.
*   `training_output/': an example of the output generated by training. it contain the best model used for the inference of this this repository.

## 4. Input Variables
The model utilizes a 2-variable feature set to estimate FCOVER:
1.  **FAPAR Average:** Fraction of Absorbed Photosynthetically Active Radiation.
2.  **Temporal Context:** Day of Year (DOY) derived from Sentinel-2 acquisition dates.

## 5. Performance Metrics
Every training run generates a `Best_Model_Performance.xlsx` file including:
*   **$R^2$ (Coefficient of Determination):** To assess goodness-of-fit.
*   **RMSE / RRMSE:** Root Mean Squared Error to quantify absolute and relative deviations.
*   **Feature Importance:** Visual bar charts indicating which satellite inputs most heavily influenced the FCOVER estimation.

## 6. Setup and Usage
Ensure the `gpp_env` Conda environment is active.

### Training the Model
Run the training script from the root of this folder:
```bash
python train_fcover.py
```

### Outputs
After execution, check the `GPP_3vars_Inference_[TIMESTAMP]` folder for:
*   The optimized **Pickle (.pkl)** model files.
*   **Regression plots** comparing observed vs. predicted FCOVER values.
